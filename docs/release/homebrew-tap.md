# Homebrew Tap Bootstrap

Shelbi's macOS package manager path uses a dedicated tap repository named
`homebrew-shelbi`. The current source repository is `jlong/shelbi`, so the
initial tap target is `jlong/homebrew-shelbi`.

User install command:

```bash
brew tap jlong/shelbi
brew install shelbi
```

## External Repository Action

Create the tap repository before the first tagged release:

```bash
gh repo create jlong/homebrew-shelbi --public --description "Homebrew tap for Shelbi"
```

Seed a fresh tap with an empty `Formula/` directory (a `.gitkeep` placeholder)
and the tap CI workflow; the established `jlong/homebrew-shelbi` repository shows
the expected layout. The first real `Formula/shelbi.rb` is generated by release
automation after GitHub Release tarballs and `checksums.txt` exist.

Set these values in `jlong/shelbi`:

- Repository variable: `HOMEBREW_TAP_REPOSITORY=jlong/homebrew-shelbi`
- Repository secret: `TAP_GITHUB_TOKEN`, scoped to open branches and pull
  requests in `jlong/homebrew-shelbi`

## Formula Shape

The release workflow runs `scripts/release/update-homebrew-formula.rb`. It
updates only `Formula/shelbi.rb`, using SHA256 values from the released
`checksums.txt`.

Supported release artifact names:

- `shelbi_Darwin_arm64.tar.gz`
- `shelbi_Darwin_x86_64.tar.gz`, only when CI builds and smoke-tests it

Each tarball must contain:

```text
shelbi
LICENSE
README.md
```

The generated formula installs `shelbi`, declares the MIT license, depends on
`tmux`, points at the GitHub release tarballs, verifies SHA256, selects arm64
vs x86_64 only when both artifacts exist, and tests `shelbi --version`.

## Unresolved Naming Decision

The source plan leaves final ownership open: `jlong/*` vs `shelbi/*`. This
bootstrap chooses `jlong/homebrew-shelbi` because the repository origin and
Cargo metadata currently point at `jlong/shelbi`.

If a `shelbi` GitHub organization becomes canonical, create
`shelbi/homebrew-shelbi`, change `HOMEBREW_TAP_REPOSITORY` to
`shelbi/homebrew-shelbi`, update package URLs to `shelbi/shelbi` if the main
repository also moves, and publish this install command instead:

```bash
brew tap shelbi/shelbi
brew install shelbi
```
